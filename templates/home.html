CurryScorer API

Welcome to the CurryScorer API

Below are the available endpoints along with example queries:

Interactive Map

Endpoint: /api/v1.0/map

This endpoint returns a JSON array of restaurant markers with details (ID, name, latitude, longitude, borough, cuisine, inspection_date) for building an interactive map using Leaflet.js. No argument is required! Click to visit!

Non-Argument Query:

GET /api/v1.0/map
Top Cuisines by Borough

Endpoint: /api/v1.0/top-cuisines

This endpoint returns the most common cuisines in a specified borough, sorted by count. It also requires an argument for data output!

Query Parameter: ?borough=[borough_name]

Example Query:

GET /api/v1.0/top-cuisines?borough=Brooklyn
Cuisine Distributions

Endpoint: /api/v1.0/cuisine-distribution

This endpoint returns the percentage distribution of different ethnic cuisines across the city, useful for building pie charts with Plotly. No argument is required! Click to visit!

Non-Argument Query:

GET /api/v1.0/cuisine-distributions
Borough Summaries

Endpoint: /api/v1.0/borough-summary

This endpoint returns summary statistics for each borough (e.g., total restaurants, average ratings, density per capita). No argument is required! Click to visit!

Non-Argument Query:

GET /api/v1.0/borough-summaries
